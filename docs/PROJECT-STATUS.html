watson-graph 금융 도메인 v2 진행

자본시장·공시(DART) GraphRAG — 현황, 왜 graph인가, 취업 데모 전략, 로드맵 · 2026-06-17

벡터(의미) + BM25(어휘)에 지식그래프(관계)를 RRF로 융합한 RAG. v2의 핵심은 엣지에 관계 타입을 부여해 멀티홉 경로 자체가 답의 근거가 되게 하는 것. 단, graph의 가치는 가정이 아니라 증명해야 할 가설이다.

1. 현 상태

백엔드 E2E 확인

/api/rag/ask가 멀티홉(사업보고서→자본시장법→파생결합증권) 정답을 출처 근거와 함께 반환. Ollama 임베딩 + ingest + RAG 살아있음. count=8(적재 전 기준).

T1-A 타입드 엔진 코드완료·빌드대기

EdgeType 9종 + 시드 관계표, 타입드 adjacency, searchWithPaths() 경로 반환. 빌드/테스트 그린은 로컬 확인 대기.

graph-stats 관측 추가

GET /api/data/graph/stats?namespace= — 노드/엣지/타입별 분포. ingest가 그래프까지 됐는지, 타입드 엣지가 생성됐는지 검증.

FE (Next) 파킹

금융 페르소나 일반 RAG 화면 스캐폴딩 완료. 기능+데이터 확정 뒤 P2에서 연결.

2. 왜 graph인가 — 정직한 분석

불편한 사실: 직전 eval에서 vector-only가 멀티홉 3/3(100%)을 찍었다. "이 코퍼스에선 vector가 안 틀렸다 = graph가 필요 없었다"는 데이터다. graph의 가치는 아직 미증명이며, 인정하고 시작한다.

graph가 정당화되는 조건 (셋 다 충족)

  • 멀티홉인데 질문↔정답 표면 겹침이 없는 질의 — 겹치면 vector가 어차피 잡아 graph 무용.
  • 하드네거티브가 깔린 코퍼스 — 같은 주제 오답이 있어야 "관계로만 도달"이 의미를 가짐.
  • 추출 가능한 이산적 관계 — 법이 분류한다 / 기관이 산출한다처럼 결정적으로 뽑혀야 함.

이 도메인 평가 — 반반

맞는 쪽

자본시장·공시는 관계가 규정에 박혀 이산적(법령→공시 의무, 법령→상품 분류, 기관→지수 산출). DART는 반정형 관계 데이터. 조건 3 충족.

불확실한 쪽

실제 사용자가 그런 멀티홉을 자주 묻나? 대부분 단일홉이고 vector가 다 잡음. 멀티홉 질문은 구성된 케이스에 가까움. 조건 1의 현실 빈도 미증명.

3. 데모 전략 (취업 데모)

목표가 취업 데모면 무게가 "프로덕션에서 graph가 이기냐"에서 "엔지니어링 판단력을 보이냐"로 옮겨간다. 면접관이 보는 건 그쪽이다.

  • (B) 설명가능성을 주연으로. 타입드 경로 시각화가 화면에서 즉시 읽힌다. vector 유사도 점수가 못 주는 "추론하는 그림". 금융 감사/근거 요구와도 맞음.
  • (A) recall은 반증 가능한 eval로 보조. 조작된 숫자 자랑은 면접에서 약하다. "언제 이기고 언제 불필요한지 안다"가 시니어 신호.
  • 핵심 아티팩트: 같은 멀티홉 질문 → vector는 오답/약함 / +graph는 정답 + 화면에 경로. 옆에 정직한 eval 표(단일홉은 graph 꺼도 vector가 이김).

서사 한 줄: "하이브리드 검색을 만들고, 반증 가능한 eval로 각 레이어가 언제 값을 하는지 안다. 그리고 타입드 경로로 답의 근거를 시각화한다."

5. T1-D 변별 픽스처 — before/after와 정직성

graph가 일할 헤드룸을 만들기 위해 코퍼스를 13개 본문으로 늘리고 하드네거티브를 깔았다. 채점은 키워드가 아니라 gold 문서 단위로 정밀화.

변별 장치

  • 하드네거티브 — 보고서 질문에 분기·감사보고서, 지수 질문에 코스피·코스닥
  • vector-hard 멀티홉 — 질문과 정답이 표면 어휘 비공유
  • 문서 단위 정밀 채점 — 오답 문서가 같은 단어 담아도 불통과

정직성(falsifiable)

  • 단일홉 대조군 6개 — graph 켜도 안 떨어져야(노이즈 체크)
  • graph가 질 수 있는 케이스 포함
  • 한 멀티홉은 두 번째 홉이 UNTYPED — 부분 타입도 작동함을 정직하게

기대 결과 (가설 · 실행으로 검증)

구간vector-only+graph
단일홉 (6)높음같음 (안 떨어져야)

멀티홉에서 +graph가 못 이기면 → "이 도메인엔 graph가 recall로 정당화 안 됨"이 정직한 결론. 그땐 기본 off + 설명가능성 용도로 포지셔닝. 설계: eval/FIXTURE-DESIGN.md
